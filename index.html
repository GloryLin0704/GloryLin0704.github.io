The Last For 2019

发表于 2019-12-31

前言

今年的故事有点特别，那就从春夏秋冬说起吧。

春

春天是播种的季节，也是开始的季节

​ 年初正式成了大三下学期的学生，摆在眼前的就不止学业那么简单了，还有找实习，为了找到满意的实习，过年的时候还是有好好准备过一番的。

​ 3月参加了春招，如果一切都像看起来那样顺利就好了，两个月下来，只面试了三家公司，最后选择了在虎牙实习；看着身边的朋友们都去了腾讯头条，有些失落，也有不甘。

​ 在这个春天里，最有意思的，就是待在创客 C-116 的那两个月，每天在复习前端基础，改简历，投简历，每天都能和战术大师吃吃鸡、吹吹水，虽然房间很小，但是日子很潇洒，那应该是今年最无忧无虑，最空闲的时候了。

夏

夏天很热，空调很凉

​ 这个夏天，大多数都是在去公司路上 -> 在公司 -> 公司回来路上，日子过得很慢，不算很忙，但也充实；在公司和头帆，P仔，豪哥组成了「广工四杰」，后来头帆跑路，头帆成了头帆，「广工四杰」成了「广工三轮车」，在豪哥这个车头的带领下，颠簸前进。现在回想，在虎牙实习的日子还是挺滋润的，有说有笑，热热闹闹。

​ 这个夏天，也是许多朋友的毕业季，去了好几个学校参加朋友的毕业典礼，也在自己学校送走了上一届的老大哥小姐姐们；最大的反差也许就是深夜里，再也没有说走就走的 M 记了，似乎有些人以后很难再见到，也许这一别，就是永远，写到这，脑海里回想起很多熟悉的面孔，但这些人早已在东南西北了。

​ 这个夏天，我也在弥补春天留下的遗憾，每天都会抽点时间来回顾一下知识，一直在准备。等待秋天的到来。

​ 在夏天快要结束的时候，实习答辩，并没有像预期那样拿到转正名额；其实也没什么，我在这实习的目的也达到了，学了不少东西，交了许多朋友；看着身边许许多多朋友在朋友圈发腾讯头条转正的结果，有些妒忌，有些羡慕，当时在想，希望秋天来的时候，我也可以，于是在夏天要结束的时候，我也结束了在虎牙的实习回到了学校，当然也带着虎牙的回忆离开：

  • 光哥骚里骚气的，和微信头像差距有点大
  • 海哥人狠话不多，很耐心教我 debug，实则也很闷骚
  • 最怕锦帆拍我椅子，这种情况大多数是又有需求了
  • 想到娜姐，不是喜茶就是一点点
  • 伟立很强，颜值和肌肉并存的男人
  • 涛哥总会在点奶茶的时候出现
  • 小姐姐很勤奋，总在加班
  • 天己 gay 里 gay 气的哈哈哈
  • 伟哥不愧是伟哥
  • 梁宇总能讲很多故事
  • 祖传 ai-docker
  • 天天和P仔在茶水间摸鱼白嫖牛奶
  • 豪哥说等等我马上就好，其实是半个小时以上
  • 想起这些事就不自觉地笑了一下

秋

秋天很忙碌，收获也很多

​ 这个秋天，让我感受到了找工作的不容易，又回到了创客，这回是在 B-121 ，和 P 仔相互扶持，每天都在牛客网上找招聘的帖子，然后发简历求内推，9月的前两周，我们都在投简历，但却没收到一个面试通知，也因此陷入恐慌，遂加大力度投简历，银行外企凡是和互联网相关的都投。

​ 这样投简历的结果是，9月的后两周，天天在面试，整个广州奔波，挤过早高峰的死亡3号线，坐过两个小时的地铁。在创客的两个月，每天和 P 仔研究面试题，查漏补缺，看看算法，然后一边看牛客网和 P 仔吐槽那些拿到 BAT offer 诚心求对比的帖子，一边不断挖掘面试亮点，总结在虎牙做过的事，想着怎样把这件事说得让面试官感兴趣。

​ 也许真的是如古话一般，皇天不负有心人。P 仔如愿以偿进了头条和头帆做同事，而我这个秋天也拿到了不少 offer：

  • 小米
  • 微众
  • 微保
  • 中信
  • 头条
  • 阿里

​ 阿里的面试是我走过最长的路，前前后后三个月，都已经放弃不抱希望了，却在最后收到了通知。阿里曾是我最开始的梦，在频频上热搜的互联网式加班，我已经有些厌倦，曾和朋友说，除非能去阿里，不然我应该会去个银行摸摸鱼过过小日子，当时也是准备去微众的了；没想到，兜兜转转，把最开始的目标实现了，这也许就是，坚持下去，时间会给你最好的答案吧。

冬

冬天有点热，但也很冷

​ 秋招告一段落之后，在学校基本没啥事做了，朋友们不是在备考就是在实习，留在学校的没几个，所以我每天在宿舍睡觉打游戏，仿佛生活失去了目标，玩了一段时间之后，感觉生活索然无味，所以决定去阿里实习一段时间，先熟悉熟悉环境。

​ 十一月底，一个人来到了杭州，在广州上飞机的时候，穿着的是短袖，下机后冷得直哆嗦，温差居然有十几度，原来这才是冬天，前面二十几年的冬天都白过了。

​ 刚来到杭州最难的是找房子，没有朋友在身边，一个人在陌生的地方，一点点摸索着租房，一点点摸索着生活。P 仔每天都和我说，一个人要学会孤独。好在，一切并没有想象中那么糟，每个人都可以被世界温柔以待。

​ 在阿里实习了一个月，的确接触到了很多不一样的东西，比如有时候感触，大家思考问题的点很有意思，经常能想到一些不一样的想法；比如饭点都在疑惑，今天去几号楼吃饭，吃啥；再比如每天都猜不到什么时候下班。

​ 习惯真是个可怕的东西，用久了 windows ，用 mac 在按键上总乱岔。

最后

去年留下的问题，现在可以回答了。

曾站在高峰，也曾跌落平地，最后一直在路上。

留给明年的问题，找到女朋友了吗？

就这样了吧，2020！顺利毕业，早点下班！

Cookie

为什么会有 Cookie

HTTP 协议是无状态的，HTTP 协议自身不对请求和响应之间的通信状态进行保存

什么是 Cookie

Cookie 指某些网站为了辨别用户身份而储存在用户本地终端上的数据(通常经过加密，如 token )。

通过给每个不同的客户端一些带有特定信息的 Cookie，就可以在每次请求的时候，服务器获取到一些期望获取的信息，比如知道这次访问的用户是谁( token )。

  • 其实 Cookie 就是一种浏览器管理状态的一个文件。

执行上下文栈

当 JavaScript 顺序执行下来的时候，会遇到三种可执行代码，全局代码，函数代码，eval代码

当 JavaScript 遇到执行到这些代码的时候，会创建一个环境，执行上下文，与此同时，会将这个环境压入一个名为执行上下文栈的环境去管理不同的执行上下文。

执行上下文(Execution context);

执行上下文栈(Execution context stack, ECS);

形象点可以模拟成一个数组 ECStack = [];

在 Javascript 开始解释代码的时候，最先遇到的是全局代码，所以全局代码的执行上下文最先入栈，globalContext.

为什么要有事件循环

JavaScript 被设计的初愿是执行在浏览器上可以动态加载的脚本，以便进行 DOM 节点的操作，所以，如果它是像 Java 一样的多线程语言，那么对 DOM 的操作就是未知且不可预测的了。

所以：

JavaScript 是一门单线程语言。

单线程就意味着有丶事情做起来没那么理想化，比如读取一个文件的事件，你以为它读完了文件内容而进行下一步的操作，然而运行的时候却发现，并没有这个结果存在，最后才发现，oh！！原来这个操作是异步的。。对，没错，单线程由此引出了 同步 和 异步 的问题。只有将某些操作(如读取文件)，作为异步执行，才能避免 阻塞 ，使脚本执行时间更迅速，用户体验更好。

缓存详解（转）

发表于 2018-02-09

缓存是一种保存资源副本并在下次请求时直接使用该副本的技术。

说实话，我起始真的不知道怎么去介绍缓存，所以引用了上面相对官方的定义。我想几乎每个开发者都碰到过缓存的问题吧，甚至有很多情况下我们会说这个问题已经修复了,你清理下缓存就好了。这篇文章我们就细细的来挖掘下缓存的种种轶事。

🦋缓存的种类

很多开发者习惯把cookie、webStorage以及IndexedDB存储的数据也称之为缓存，理由是都是保存在客户端的数据，没有什么区别。其实这是不严谨的，cookie的存在更多的是为了让服务端区别用户，webStorage和IndexedDB则更多用在保存具体的数据和在客户端存储大量结构化数据(文件/blobs)上面。

实际上所谓的缓存只有一种——它是请求资源的副本。试想一下，如果每一个资源我们客户端都会保存一份副本，这会怎么样？客户端会炸掉，开发者会疯掉！所以我们需要一份协议来处理缓存，可以让开发者控制缓存的建立和删除。谁呢？还能有谁，HTTP呗。HTTP协议里定义了很多关于缓存的请求和响应字段，这也是接下来我们重点要逼逼叨的对象，研究下究竟是哪些字段怎么影响缓存的。
